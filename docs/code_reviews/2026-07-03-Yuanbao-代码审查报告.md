### 问题 2：裸 `except:` 子句捕获所有异常

**文件**：`llmakits/message/formatter.py`
**行号**：`98`、`104`
**级别**：🔴 严重bug

**问题描述**：
`convert_to_json()` 函数中使用了两个裸 `except:` 语句，会无差别地捕获所有异常（包括 `KeyboardInterrupt`、`SystemExit`、`MemoryError` 等严重异常），导致程序无法正常退出或无法感知致命错误。Python 最佳实践要求捕获具体异常类型。此外，第一次裸 `except:` 还使用了 `eval()`——这是一种潜在的安全风险（代码注入攻击），且在生产环境中应严格避免。

**代码片段**：
```python
def convert_to_json(text: str) -> Any:
    ...
    try:
        if processed_text.startswith("```json"):
            processed_text = processed_text.strip("```json\n")
        converted_json = json.loads(processed_text)
    except json.JSONDecodeError:
        try:
            converted_json = eval(processed_text)  # ← 使用 eval()，安全风险
        except:  # ← 裸 except，捕获所有异常
            ...
```

```python
        except:  # ← 行104，裸 except
            try:
                extracted_json = extract_json_from_string(processed_text)
                ...
```

**修复建议**：
```python
    except json.JSONDecodeError:
        try:
            converted_json = eval(processed_text)  # 建议用 ast.literal_eval 替代 eval
        except (NameError, SyntaxError, ValueError):  # ← 仅捕获预期异常
            try:
                extracted_json = extract_json_from_string(processed_text)
                if extracted_json:
                    converted_json = json.loads(extracted_json)
            except (json.JSONDecodeError, ValueError)  # ← 限制异常类型
```

### 问题 3：裸 `except:` 导致异常处理静默失败

**文件**：`llmakits/message/formatter.py`
**行号**：`104`（第二次裸 `except:` 块内）
**级别**：🔴 严重bug

**问题描述**：
第二个裸 `except:` 块中，`eval` 失败后尝试 `extract_json_from_string`，若后者也抛出任何异常（包括非 JSON 相关异常），异常将被静默吞掉，不打印任何错误信息，函数最终走向 `raise Exception("format_json_error,无法解析为json格式")`，掩盖了真正的根因。这会严重干扰问题排查。

**代码片段**：
```python
            except:
                try:
                    extracted_json = extract_json_from_string(processed_text)
                    if extracted_json:
                        converted_json = json.loads(extracted_json)
                except:  # ← 异常被吞掉，无日志
                    pass
```

**修复建议**：
添加明确的异常类型和日志记录：

```python
            except Exception as inner_e:
                try:
                    extracted_json = extract_json_from_string(processed_text)
                    if extracted_json:
                        converted_json = json.loads(extracted_json)
                except Exception as second_e:
                    print(f"JSON解析尝试均失败: eval→{inner_e}, extract→{second_e}")
```
